# Scapy Packet Crafting & Network Traffic Analysis Lab

## Overview

This repository contains a cybersecurity laboratory focused on **network packet manipulation and traffic analysis** using industry-standard tools such as **Scapy, tcpdump, and Wireshark**.

The goal of this lab is to understand how attackers and penetration testers interact with network traffic by crafting custom packets and analyzing captured data. The exercise demonstrates how reconnaissance can be performed against a target host and how sensitive information may be exposed in insecure network communications.

The report included in this repository explains the process of generating packets, capturing traffic, and analyzing the resulting data.

---

## Objectives

The main objectives of this laboratory are:

- Learn how to use **Scapy** to craft and send custom network packets.
- Perform **network reconnaissance** using ICMP and TCP packets.
- Capture traffic using **tcpdump**.
- Analyze captured packets using **Wireshark**.
- Understand how credentials and session information can be exposed in network traffic.

---

## Tools Used

The following tools were used during the laboratory:

- **Scapy** – Python-based packet manipulation framework  
- **tcpdump** – Command-line packet capture tool  
- **Wireshark** – Graphical packet analysis tool  
- **Linux (Kali)** – Testing environment

## Laboratory Activities

### 1. Exploring Protocols with Scapy

The laboratory begins by launching Scapy from the terminal and exploring available network protocol layers using the `ls()` command. This allows inspection of protocols supported by Scapy, including TFTP and IP.

---

### 2. IPv4 Header Analysis

The structure of the IPv4 header is examined, including important fields such as:

- Version
- Differentiated Services (DSCP)
- Time To Live (TTL)
- Protocol Identifier
- Header Checksum
- Source and Destination IP addresses

Understanding these fields is essential for crafting valid packets.

---

### 3. Packet Sniffing with Scapy

Network traffic is captured using the `sniff()` function in Scapy.

Captured packets are analyzed using summary functions that display protocol layers such as:

- Ethernet
- IPv4
- ICMP

Traffic generated by **ping requests** is used to demonstrate packet capture.

---

### 4. Creating Custom ICMP Packets

Scapy is used to generate a custom ICMP packet containing a payload message.

Example command:

```python
send(IP(dst="10.6.6.23")/ICMP()/"This is a test")
```

This demonstrates how attackers or testers can generate network traffic manually.

---

### 5. TCP SYN Packet Creation

A TCP SYN packet is crafted to test whether a specific port is open on the target host.

Example command:

```python
send(IP(dst="10.6.6.23")/TCP(dport=445,flags="S"))
```

If the port is open, the host responds with a **SYN-ACK packet**, indicating that the service is available.

This technique is commonly used in **port scanning and reconnaissance**.

---

### 6. Capturing Traffic with tcpdump

Traffic is captured using tcpdump and saved into a `.pcap` file.

Example command:

```
sudo tcpdump -i eth0 -s 0 -w packetdump.pcap
```

Parameters used:

- `-i` specifies the network interface  
- `-s 0` captures the full packet  
- `-w` writes the output to a file  

---

### 7. Traffic Analysis with Wireshark

The captured `.pcap` file is analyzed using Wireshark.

Filters are applied to identify specific types of traffic such as:

- DNS queries
- HTTP requests
- POST requests

This allows inspection of communication between the client and servers.

---

### 8. Credential Interception

An insecure login request is generated on a test server.

Using Wireshark filters, HTTP POST requests are identified and analyzed. The captured packets reveal the transmitted **username and password**, demonstrating how credentials can be exposed when encryption is not used.

Session cookies such as **PHPSESSID** are also visible in the traffic.

These cookies can potentially be used in **session hijacking attacks**.
